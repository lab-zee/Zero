## Open Questions Requiring Decisions

### 1. Answer Mode & Image Generation Coupling

**Question**: Should answer mode affect image generation frequency?

**Options**:
- **A**: Summary mode = No images (text only)
- **B**: Summary mode = Reduced images (only critical visualizations)
- **C**: Answer mode doesn't affect image generation (agent decides independently)

**Impact**:
- Option A: Maximum cost savings, may sacrifice clarity
- Option B: Balanced approach, still reduces costs
- Option C: Maintains quality, less cost optimization

**Recommendation**: Option B (reduced images in summary mode)

**Decision**: ✓ Option B - Summary mode uses reduced images (only critical visualizations)

**Decided By**: User / 2026-01-23

### 2. Website Scraping Cache Duration

**Question**: How long should we cache website scraping results?

**Context**: If a user tries to edit an organization shortly after creation, should we re-scrape or use cached data?

**Options**:
- **A**: No caching (always scrape fresh)
- **B**: 24 hours cache
- **C**: 7 days cache
- **D**: Cache indefinitely with manual refresh option

**Impact**:
- Option A: Always current, but slower and more API calls
- Option B: Good balance between freshness and performance
- Option C: Better performance, may show outdated data
- Option D: Best performance, gives user control

**Recommendation**: Option D (indefinite cache with refresh button)

**Decision**: ✓ Option C - 7 days cache

**Decided By**: User / 2026-01-23

### 3. Follow-up Question Generation Ownership

- Option D: Fastest, may be lower quality

**Decision**: ✓ Option C - Dedicated agent (specialized in question generation)

### 4. Citation Format Standard

**Question**: Which citation format should we use?

**Options**:

### 5. Clarification Question Limit

**Question**: Maximum number of clarification questions to ask before proceeding anyway?

**Context**: Don't want to block users with too many questions, but need sufficient information.

**Options**:
- **A**: 3 questions maximum
- **B**: 5 questions maximum
- **C**: No limit (ask as many as needed)
- **D**: Progressive (3 questions, then offer "proceed anyway" option)

**Impact**:
- Options A-B: User-friendly, may not gather all needed info
- Option C: Thorough, may frustrate users
- Option D: Balanced, gives user control

**Recommendation**: Option D (progressive approach)

**Decision**: ✓ Option D - Progressive (3 questions, then offer "proceed anyway" option)

**Decided By**: User / 2026-01-23

### 8. Org Onboarding: Manual Entry Fallback

**Question**: If website scraping fails, should we require manual entry or make it optional?

**Options**:
- **A**: Required (block creation until all fields filled)
- **B**: Optional (allow empty fields, can edit later)
- **C**: Smart required (only essential fields required)
- **D**: Progressive (require fields based on usage)

**Impact**:
- Option A: Complete data, friction in onboarding
- Option B: Easy onboarding, incomplete data
- Option C: Balanced, needs definition of "essential"
- Option D: Best UX, complex implementation

**Recommendation**: Option C (name, industry required; rest optional)

**Decision**: ✓ Option C - Smart required (only essential fields required: name, industry)

**Decided By**: User / 2026-01-23

### 9. Answer Mode Storage

**Question**: Should answer mode be stored per-query or just per-thread?

**Context**: Thread has default_answer_mode. Should individual queries also store their mode?

**Options**:
- **A**: Thread only (queries inherit, no override storage)
- **B**: Query stores actual mode used (even if same as thread default)
- **C**: Query stores only if different from thread default

**Impact**:
- Option A: Minimal storage, lose history of overrides
- Option B: Complete audit trail, slight storage overhead
- Option C: Efficient storage, maintains override history

**Recommendation**: Option B (store per-query for full history)

**Decision**: ✓ Option B - Query stores actual mode used (full audit trail)

**Decided By**: User / 2026-01-23

### 10. Tab Persistence

**Question**: Should the selected tab persist across messages or reset to Summary for each new message?

**Options**:
- **A**: Reset to Summary for each message
- **B**: Persist last selected tab within session
- **C**: Persist per-user preference in settings
- **D**: Smart persistence (Summary for new questions, last tab for same thread)

**Impact**:
- Option A: Consistent, predictable, may annoy power users
- Option B: Convenient, may confuse if user forgets setting
- Option C: Flexible, adds preference complexity
- Option D: Best of both, harder to implement

**Recommendation**: Option A (always reset to Summary)

**Decision**: ✓ Option A - Reset to Summary for each message

**Decided By**: User / 2026-01-23
